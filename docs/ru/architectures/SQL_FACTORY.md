# SQL Factory (SQL-фабрика)

## Обзор

Многораундовая генерация с фильтрацией качества и расширением по похожести. Итеративно строит пул SQL-кандидатов через эксплуатацию (улучшение хороших) и исследование (генерацию разнообразных вариантов).

## Алгоритм

1. **Warmup** (1 раунд) — сгенерировать начальную партию, выполнить, построить пул
2. **Main Loop** (до 3 раундов):
   - **Exploitation** — сгенерировать `gen_batch` кандидатов, направляемых лучшими из пула
   - **Exploration** — сгенерировать `exp_batch` вариаций, отфильтровать по похожести
   - **Score** каждого кандидата: exec_ok + бонус за покрытие таблиц − штраф за похожесть
   - **Pool update** — оставить топ-`target_pool_size` кандидатов
3. **Условия остановки:**
   - Насыщение (нет новых добавлений N раундов)
   - Превышен временной бюджет
   - Стагнация лучшей оценки
4. **Select** — выбрать лучшего из финального пула

## Оценка (scoring)

- База: 1.0 если exec_ok, иначе 0.0
- Бонус за таблицы: +0.05 на уникальную таблицу (cap 6)
- Штраф за похожесть: если Jaccard > 0.85 к членам пула
- Бонус за сложность: +0.05 до +0.2 для сложных запросов

## Параметры

```yaml
architecture:
  name: sql_factory
  params:
    max_rounds: 3
    warmup_rounds: 1
    gen_batch: 2
    exp_batch: 2
    target_pool_size: 5
    sim_threshold: 0.85
    time_budget_per_task_sec: 40
    parallelism: parallel
    max_workers: 5
```

| Параметр | По умолчанию | Описание |
|-----------|---------|-------------|
| `max_rounds` | 3 | Макс. раундов генерации |
| `gen_batch` | 2 | Кандидатов на эксплуатацию за раунд |
| `exp_batch` | 2 | Кандидатов на исследование за раунд |
| `target_pool_size` | 5 | Размер пула |
| `sim_threshold` | 0.85 | Порог дедупликации по похожести |
| `time_budget_per_task_sec` | 40 | Временной лимит на задачу |

## Метрики похожести

- **Token Jaccard** — `|A ∩ B| / |A ∪ B|` на множествах токенов SQL
- **AST similarity** — структурное сравнение через дерево разбора
- Объединены с настраиваемыми весами (по умолчанию: 0.6 tok + 0.3 ast + 0.1 emb)

## CLI

```bash
uv run python -m evalsuite run --model qwen3-coder-next --bench bird_sqlite --architecture sql_factory
```
