# SGR (Semantic Graph Reasoning)

## Обзор

Обоснование схемы + структурный план + K генераций SQL + починка. Перед генерацией SQL модель анализирует вопрос и определяет релевантные таблицы, колонки и соединения — снижая галлюцинации.

## Алгоритм

1. **Grounding** — LLM выделяет из схемы релевантные таблицы, колонки, соединения, фильтры
2. **Plan** — декомпозиция запроса: агрегации, GROUP BY, фильтры, ORDER BY
3. **SQL Synthesis** (K=3 кандидата) — генерация SQL с использованием результатов grounding + plan как контекста
4. **Preflight + Execution** — валидация и исполнение каждого кандидата
5. **Repair** (опционально) — если кандидат упал с `pred_bind_fail` или `pred_runtime_fail`, повтор с текстом ошибки
6. **Scoring** — базовая оценка 1.0 при exec_ok; штраф −0.2 за каждую таблицу вне grounding; штраф −0.1 если SQL длиннее 2000 символов

## Параметры

| Параметр | По умолчанию | Описание |
|-----------|---------|-------------|
| `num_candidates` | 3 | Число SQL-кандидатов |
| `generation_timeout` | 15s | Таймаут на кандидата |
| `MAX_SCHEMA_CHARS` | 12000 | Лимит длины схемы |

## Починка (repair)

Срабатывает при ошибках: `pred_bind_fail`, `pred_runtime_fail`. Упавший SQL и текст ошибки отправляются обратно в LLM. Одна попытка починки на кандидата.

## CLI

```bash
uv run python -m evalsuite run --model qwen3-coder-next --bench bird_sqlite --architecture sgr
```

## Результаты

Для qwen3-coder-next, `context_mode=toolchain`:

| Бенчмарк | Accuracy | Failures | Ср. латентность |
|----------|----------|----------|-----------------|
| BIRD | 30.1% (461/1534) | 108 | 16.8s |
| Spider2 | 67.4% (83/123) | 16 | 29.2s |
| TPC-DS | 4.0% (4/99) | 44 | 22.0s |

Лучший результат на Spider2 среди одиночных архитектур (до Hybrid). Grounding значительно помогает на задачах с большими схемами. На TPC-DS SGR проигрывает из-за плейсхолдеров — модель на аналитическом домене оставляет параметры нерезрешёнными.
